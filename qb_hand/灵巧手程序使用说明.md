# 灵巧手控制程序使用说明

本说明对应新版 `dexterous_hand_controller.py`。程序已取消命令行子命令和交互式控制台，改为在主程序的 `PROGRAM_CONFIG` 中选择要执行的功能。

## 1. 使用前准备

安装串口依赖：

```bash
pip install pyserial
```

确认硬件连接：

- 使用 USB 转 TTL 串口、蓝牙、WIFI 或 Ethernet 时，本程序按《灵巧手-通信协议-V1.5》的串口/WIFI/Ethernet 协议发帧。
- 默认波特率为 `115200`，8 个数据位，1 个停止位，无奇偶校验。
- Windows 串口通常是 `COM3`、`COM4` 这类名称，实际端口以设备管理器为准。

## 2. 运行方式

打开 `dexterous_hand_controller.py`，找到文件下方的配置：

```python
PROGRAM_CONFIG = HandProgramConfig(
    port="COM3",
    baudrate=115200,
    timeout=0.2,
    action="open",
    move_angles=(0, 0, 0, 0, 0, 0),
    dofs=None,
    tactile_timeout=2.0,
    sequence_delay=1.0,
    force_wait_timeout=8.0,
    wait_force_ack=False,
    use_force_status_query=False,
    force_run_seconds=3.0,
    init_before_force=True,
    use_official_force_config=False,
)
```

修改 `port` 和 `action` 后直接运行：

```bash
python dexterous_hand_controller.py
```

## 3. 可选功能

`action` 支持以下值：

| action | 作用 | 是否等待设备回执 |
| --- | --- | --- |
| `open` | 全部自由度回到 0 度，五指张开 | 否 |
| `thumb_open` | 官方示例 HandOpen 动作 | 否 |
| `grasp` | 安全小角度抓握动作 | 否 |
| `one` | 官方示例 HandOne 动作 | 否 |
| `move` | 按 6 个自由度角度运动 | 否 |
| `stop` | 急停全部或指定自由度 | 否 |
| `resume` | 恢复全部或指定自由度 | 否 |
| `angles` | 查询当前角度 | 是 |
| `force_demo` | 下发默认力控参数 | 默认不等待 |
| `force_set` | `force_demo` 的别名，下发默认力控参数 | 默认不等待 |
| `force_official_demo` | 下发与 `hand.py` 一致的官方力控参数 | 默认不等待 |
| `force_official_set` | `force_official_demo` 的别名 | 默认不等待 |
| `force_start` | 启动力控 | 默认不等待 |
| `force_stop` | 退出力控并回 0 度 | 默认不等待 |
| `force_status` | 查询力控执行状态 | 是 |
| `tactile_once` | 读取一帧主动上传触觉数据 | 是，等待触觉帧 |
| `force_grasp_sequence` | 设置力控参数、启动、等待完成、退出力控 | 是 |
| `force_start_sequence` | 同一次运行内下发安全力控参数并启动，不查询状态 | 默认不等待 |
| `force_official_start_sequence` | 同一次运行内下发官方力控参数并启动，不查询状态 | 默认不等待 |
| `motion_demo_sequence` | 连续执行多个普通控制功能的示例 | 部分命令无回执 |
| `force_control_example` | 完整力控模式示例 | 是 |

## 4. 角度控制

6 个自由度顺序固定为：

1. `thumb_rotate`：拇指旋转/内旋
2. `thumb_bend`：拇指弯曲
3. `index`：食指弯曲
4. `middle`：中指弯曲
5. `ring`：无名指弯曲
6. `little`：小指弯曲

例：让拇指旋转 60 度、拇指弯曲 20 度，其余手指弯曲 30 度：

```python
PROGRAM_CONFIG = HandProgramConfig(
    port="COM3",
    action="move",
    move_angles=(60, 20, 30, 30, 30, 30),
)
```

程序会检查角度必须在 `0~90` 度内。

内置 `grasp` 已从官方大角度示例改成较保守的 `(45, 25, 35, 35, 30, 30)`，用于降低首次测试时的碰撞风险。如果仍有干涉，请继续降低对应自由度角度。

`thumb_open` 和 `one` 会按官方 `hand.py` 的原始字节发包：`HandOpen` 使用 `0x75`，`HandOne` 使用 `0x55`。自定义 `move` 仍限制在 `0~90`，避免误输入过大的目标值。

## 5. 急停和恢复

急停全部自由度：

```python
PROGRAM_CONFIG = HandProgramConfig(
    port="COM3",
    action="stop",
    dofs=None,
)
```

只急停食指和中指：

```python
PROGRAM_CONFIG = HandProgramConfig(
    port="COM3",
    action="stop",
    dofs=("index", "middle"),
)
```

恢复食指和中指：

```python
PROGRAM_CONFIG = HandProgramConfig(
    port="COM3",
    action="resume",
    dofs=("index", "middle"),
)
```

自由度别名也可使用 `tr`、`tb`、`i`、`m`、`r`、`l`。

## 6. 力控流程

按协议推荐流程，力控通常分为：

1. `force_demo`：下发力控参数。
2. `force_start`：启动力控，手指先运行到初始角度，再开始抓取。
3. `force_status` 或 `angles`：执行过程中查询状态或角度。
4. `force_stop`：退出力控，各自由度回到 0 度。

可以直接使用完整流程：

```python
PROGRAM_CONFIG = HandProgramConfig(
    port="COM3",
    action="force_grasp_sequence",
)
```

注意：手册说明开启力控后 `0x10` 普通角度运动命令无效。执行完力控后要先 `force_stop` 解锁，再进行普通角度控制。

程序已对 `4B` 力控状态回复做了容错处理：如果设备返回的 D2 帧长与实际 D4~D9 状态字节长度不一致，会继续读取到 `0x5D` 帧尾后再校验和解析。

`force_set` 和 `force_demo` 等价，都是下发较保守的 `default_force_grasp_config()`。`force_official_set` 和 `force_official_demo` 会下发与官方 `hand.py` 中 `FC` 字节一致的力控参数。

默认 `wait_force_ack=False`，行为与官方 `hand.py` 一样：力控设置、启动、停止只发送，不等待回执。如果你的设备确认会返回回执，可以改成 `wait_force_ack=True`，此时程序会返回 `status`、`ok` 和 `raw_hex`。

如果 `force_status` 没有任何回包，请不要把它当作启动失败的唯一依据。先用下面这个动作，它会在同一次串口打开期间连续发送 `40` 参数和 `4A` 启动，最接近官方 `hand.py` 的使用方式：

```python
PROGRAM_CONFIG = HandProgramConfig(
    port="COM3",
    action="force_official_start_sequence",
    sequence_delay=0.2,
    wait_force_ack=False,
)
```

如果这个动作能启动，而单独运行 `force_official_set` 后再运行 `force_start` 不能启动，说明设备在两次 Python 进程重新打开串口时丢失了力控参数或状态。

## 7. 连续使用功能示例

如果想一次运行多个普通控制功能，可以把 `action` 改成：

```python
PROGRAM_CONFIG = HandProgramConfig(
    port="COM3",
    action="motion_demo_sequence",
    sequence_delay=1.0,
)
```

该示例会按顺序执行：

1. `open`：全部张开。
2. `move([20, 0, 0, 0, 0, 0])`：小角度测试拇指旋转。
3. `move([20, 10, 20, 20, 10, 10])`：小角度抓握。
4. `stop(("index", "middle"))`：急停食指和中指。
5. `resume(("index", "middle"))`：恢复食指和中指。
6. `open`：回到张开状态。

`sequence_delay` 是每一步之间的等待时间，单位是秒。第一次测试建议保留 `1.0` 或更大，确认没有机械干涉后再缩短。

也可以在 Python 里自己写连续调用：

```python
from dexterous_hand_controller import DexterousHand
import time

with DexterousHand(port="COM3") as hand:
    print("TX:", hand.dump_frame_hex(hand.preset_open()))
    time.sleep(1)

    print("TX:", hand.dump_frame_hex(hand.move([20, 0, 0, 0, 0, 0])))
    time.sleep(1)

    print("TX:", hand.dump_frame_hex(hand.move([20, 10, 20, 20, 10, 10])))
    time.sleep(1)

    print("TX:", hand.dump_frame_hex(hand.emergency_stop(("index", "middle"))))
    time.sleep(1)

    print("TX:", hand.dump_frame_hex(hand.emergency_resume(("index", "middle"))))
    time.sleep(1)

    print("TX:", hand.dump_frame_hex(hand.preset_open()))
```

## 8. 力控模式示例

推荐先用内置示例：

```python
PROGRAM_CONFIG = HandProgramConfig(
    port="COM3",
    action="force_control_example",
    use_force_status_query=False,
    force_run_seconds=3.0,
    init_before_force=True,
)
```

该示例会按协议流程执行：

1. `preset_open()`：先发送初始化张开帧，对齐官方 `hand.py` 初始化行为。
2. `set_force_control(default_force_grasp_config())`：下发默认力控参数。
3. `force_start()`：启动力控。
4. 等待 `force_run_seconds` 秒。
5. `force_stop()`：退出力控并回到 0 度。

如果安全参数下看不到明显动作，可以临时改成官方力控参数对照测试：

```python
PROGRAM_CONFIG = HandProgramConfig(
    port="COM3",
    action="force_control_example",
    use_official_force_config=True,
    init_before_force=True,
    force_run_seconds=3.0,
)
```

官方参数中部分最大角度较大，确认不会碰撞后再测试。

你的设备目前对 `4B force_status` 没有回包，所以默认 `use_force_status_query=False`。如果确认设备支持状态查询，再改成 `True`，程序才会调用 `wait_until_force_idle()` 和 `query_angles()`。

默认力控参数也改成了低速、小角度：拇指旋转最大 35 度，拇指弯曲最大 25 度，食指/中指最大 30 度，无名指/小指最大 25 度。确认安全后再逐步增大。

也可以在 Python 里手动写：

```python
from dexterous_hand_controller import DexterousHand, default_force_grasp_config
import time

with DexterousHand(port="COM3", timeout=0.2) as hand:
    cfg = default_force_grasp_config()

    print("set:", hand.set_force_control(cfg))
    print("start:", hand.force_start())
    time.sleep(3.0)
    print("stop:", hand.force_stop())
```

上面三个写命令默认不等待回执，和官方 `hand.py` 一致。如需严格等待回执，可以写成 `hand.force_start(wait_ack=True)`。

## 9. 触觉数据

触觉数据是设备主动上传帧，不是主机查询帧。读取一帧触觉数据：

```python
PROGRAM_CONFIG = HandProgramConfig(
    port="COM3",
    action="tactile_once",
    tactile_timeout=2.0,
)
```

返回结果包含：

- `finger_id`：1 到 5，对应拇指、食指、中指、无名指、小指。
- `points`：16 个 16 位无符号触觉数据点。
- `raw_hex`：原始帧数据，便于调试。

## 10. 调试建议

第一次调试建议按这个顺序改 `action` 逐步运行：

1. `open`
2. `move`，角度先设置小一些，例如 `(20, 0, 0, 0, 0, 0)`
3. `angles`
4. `stop`
5. `resume`
6. `force_demo`
7. `force_start`
8. `force_status`
9. `force_stop`
10. `tactile_once`
11. `motion_demo_sequence`
12. `force_control_example`

拇指旋转后，拇指和食指再弯曲时要先用小角度试验，避免机械干涉。用户手册也提醒，不能用手强行掰动处于锁定状态的手指。
